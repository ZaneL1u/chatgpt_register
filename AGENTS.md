# Language Policy

This repository requires Chinese-first responses.

## Required

- All Codex replies, plans, summaries, reviews, explanations, and status updates must be written in Simplified Chinese.
- All GSD workflow outputs must be written in Simplified Chinese.
- New planning documents under `.planning/` should default to Simplified Chinese unless the user explicitly requests another language.

## Allowed Exceptions

- Keep code, commands, file paths, environment variable names, API field names, protocol names, log fragments, and third-party identifiers in their original form.
- If the user explicitly requests another language for a specific output, follow that request for that output only.

## Style

- Do not switch to English for headings or boilerplate unless there is a concrete technical reason.
- When mixed language is unavoidable, keep the explanatory text in Simplified Chinese and limit English to the smallest necessary surface.
