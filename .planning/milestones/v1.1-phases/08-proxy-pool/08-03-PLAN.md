  <name>Task 1: 向导多代理输入模式</name>
  <files>chatgpt_register/wizard.py, tests/test_wizard_proxy.py</files>
  <behavior>
    - _ask_proxies(prefill=[]) 无默认值时显示 4 种模式选择
    - _ask_proxies(prefill=["http://h:80"]) 有默认值时显示当前代理摘要
    - 单代理模式：输入 "http://h:80" → 返回 ["http://h:80"]
    - 单代理模式：输入空字符串 → 返回 []
    - 多行模式：输入多行 → 返回解析后的有效代理列表
    - 文件模式：输入 .txt 路径 → 返回文件中解析的代理列表
    - 跳过模式 → 返回 []
    - _ask_registration_config 返回 dict 含 proxies 键（list[str]）和 proxy 键（str，向下兼容）
  </behavior>
  <action>
1. **新增 `_ask_proxies(prefill: list[str]) -> list[str] | None` 函数：**
   - 导入 `from chatgpt_register.core.proxy_parser import parse_proxy, parse_proxies, parse_proxies_from_file, summarize_proxies`
   - 如果 prefill 非空，显示当前配置：`console.print(f"  当前代理: {summarize_proxies(prefill)}")`
   - 用 `questionary.select()` 提供选择：
     ```python
     choices = [
         "单个代理 (直接输入地址)",
         "多个代理 (逐行输入)",
         "从文件导入 (.txt)",
         "不使用代理",
     ]
     ```
   - **单代理模式：** `questionary.text("代理地址", default=prefill[0] if prefill else "")` + `parse_proxy()` 校验
   - **多行模式：** 循环调用 `questionary.text("代理地址 (空行结束)", default="")` 直到空输入，每行用 `parse_proxy()` 即时校验，无效行打印警告
   - **文件模式：** `questionary.text("代理列表文件路径 (.txt)")` + `parse_proxies_from_file()` + 自动识别输入是否为文件路径
   - **跳过：** 返回 `[]`
   - 输入完成后显示摘要：`console.print(f"  ✓ {summarize_proxies(proxies)}")`
   - 返回 None 表示用户取消（Ctrl+C）

2. **修改 `_ask_registration_config()` 函数：**
   - 删除现有单行代理输入代码（`proxy = questionary.text("代理地址 (留空跳过)", ...)`）
   - 替换为调用 `_ask_proxies(prefill.get("proxies", []))`
   - 返回的 dict 中：
     - `"proxies": proxies_list`（新字段）
     - `"proxy": proxies_list[0] if proxies_list else ""`（向下兼容）
   - `if None in (total_accounts, workers, proxies_result):` 检查取消

3. **测试 `tests/test_wizard_proxy.py`：**
   - 使用 `unittest.mock.patch` mock `questionary.select` 和 `questionary.text` 的 `.ask()` 返回值
   - 测试各模式下 `_ask_proxies()` 的返回值
   - 测试 `_ask_registration_config()` 返回 dict 含 proxies 键

先写测试，再修改 wizard.py。
