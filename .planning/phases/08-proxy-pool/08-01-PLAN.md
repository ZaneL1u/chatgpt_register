  <name>Task 1: 代理地址解析模块 proxy_parser.py</name>
  <files>chatgpt_register/core/proxy_parser.py, tests/test_proxy_parser.py</files>
  <behavior>
    - parse_proxy("socks5://user:pass@host:1080") 返回 "socks5://user:pass@host:1080"
    - parse_proxy("http://host:8080") 返回 "http://host:8080"
    - parse_proxy("host:8080") 返回 "http://host:8080"（默认 http）
    - parse_proxy("socks4://host:1080") 返回 "socks4://host:1080"
    - parse_proxy("ftp://host:21") 返回 None（不支持的协议）
    - parse_proxy("") 返回 None
    - parse_proxy("   ") 返回 None
    - parse_proxy("not-a-url") 返回 None（无 port）
    - parse_proxies(["socks5://h:1080", "", "bad", "http://h:80"]) 返回 ["socks5://h:1080", "http://h:80"] + 2 个警告
    - parse_proxies_from_file("proxies.txt") 读取文件每行解析，跳过空行和注释行（#开头）
    - summarize_proxies(["socks5://h:1", "http://h:2", "socks5://h:3"]) 返回 "2 个 SOCKS5 + 1 个 HTTP"
  </behavior>
  <action>
创建 `chatgpt_register/core/proxy_parser.py`，包含：

1. `SUPPORTED_SCHEMES = {"http", "https", "socks4", "socks5"}`

2. `parse_proxy(raw: str) -> str | None`：
   - strip 后空字符串返回 None
   - 无 `://` 时补 `http://` 前缀
   - 用 `urllib.parse.urlparse` 解析
   - scheme 不在 SUPPORTED_SCHEMES 中返回 None
   - 无 hostname 返回 None
   - 返回标准化后的 URL 字符串

3. `parse_proxies(lines: list[str]) -> tuple[list[str], list[str]]`：
   - 返回 (valid_proxies, warnings)
   - 对每行调用 parse_proxy，有效的加入列表，无效的生成警告消息

4. `parse_proxies_from_file(path: str) -> tuple[list[str], list[str]]`：
   - 读取文件，跳过空行和 # 开头的注释行
   - 调用 parse_proxies 处理

5. `summarize_proxies(proxies: list[str]) -> str`：
   - 按协议分组统计，返回如 "2 个 SOCKS5 + 1 个 HTTP" 格式的摘要
   - 空列表返回 "无代理"

先写测试，再实现。TDD RED→GREEN→REFACTOR。
