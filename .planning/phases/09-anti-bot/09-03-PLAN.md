**batch.py — 改造 worker 提交循环:**

在 `run_batch()` 函数的 `with ThreadPoolExecutor(max_workers=actual_workers) as executor:` 块中，找到现有的 worker 提交循环：

```python
futures = {}
for idx in range(1, total_accounts + 1):
    future = executor.submit(
        _register_one, idx, total_accounts, config, output_file, _print_lock, _file_lock, dashboard, proxy_pool
    )
    futures[future] = idx
```

改为带错开延迟的版本：

```python
futures = {}
for idx in range(1, total_accounts + 1):
    future = executor.submit(
        _register_one, idx, total_accounts, config, output_file, _print_lock, _file_lock, dashboard, proxy_pool
    )
    futures[future] = idx
    # 错开启动：第一个立即启动，后续 worker 间隔 2-8s（正态分布）
    if idx < total_accounts:
        stagger = max(2.0, min(8.0, random.gauss(5.0, 1.5)))
        print(f"  ◆ Worker {idx} 已提交，等待 {stagger:.1f}s 后提交下一个...")
        time.sleep(stagger)
```

需要在文件顶部确认 `import random` 已存在（当前未导入，需添加）。

```python
class TestWorkerStagger:
    def test_worker_stagger_distribution(self):
        """ANTI-05: stagger 延迟在 2-8s 范围内。"""
        import random as _random
        for _ in range(100):
            val = max(2.0, min(8.0, _random.gauss(5.0, 1.5)))
            assert 2.0 <= val <= 8.0

    @patch("chatgpt_register.core.batch.time.sleep")
    @patch("chatgpt_register.core.batch.random.gauss", return_value=5.0)
    def test_worker_stagger_delay(self, mock_gauss, mock_sleep):
        """ANTI-05: worker 提交间调用 time.sleep 进行错开。"""
        # 验证 random.gauss 和 time.sleep 在 batch 模块中可被 mock
        from chatgpt_register.core import batch
        assert hasattr(batch, 'random')
        assert hasattr(batch, 'time')
```

注意：完整的集成测试（实际运行 run_batch 并验证 sleep 调用）过于复杂，需要 mock 大量依赖。这里只验证 stagger 分布逻辑和模块 import 正确性。实际验证通过运行时日志观察。
